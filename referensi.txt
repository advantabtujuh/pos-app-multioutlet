Rekomendasi Arsitektur Database (Best & Secure)
1. Dual Database per Outlet
SQLite (Local di mesin outlet) → Untuk transaksi harian (penjualan ecer/grosir, retur, hutang member). Sangat cepat & tidak butuh koneksi.
MariaDB Local (di setiap outlet) → Untuk mutasi stok, sync data master (produk, harga, customer), dan reporting ke pusat.
MariaDB Central (di server pusat) → Master database, aggregator semua outlet.

Komponen			Database			Koneksi Via			Tujuan
--------------------------------------------------------------------------------------
Outlet (Kasir)		SQLite				Local file			Transaksi cepat & offline
Outlet				MariaDB Local		localhost 			Mutasi & sync
Pusat				MariaDB Central		Local				Master data
Pusat ke Outlet		MariaDB Outlet		Tailscale IP		Monitoring realtime

Sync Strategy (Realtime-ish)
Outlet akan push data penting (transaksi, mutasi terima, stok opname) ke MariaDB Local-nya.
Pusat akan pull data dari MariaDB tiap outlet via Tailscale IP (polling setiap 5-10 detik atau pakai long polling).
Untuk mutasi stok: Pusat tulis ke MariaDB Outlet target langsung via Tailscale.
Master data (produk, harga, customer) sync dari Pusat ke Outlet (bidirectional).

Pusat punya list semua outlet + Tailscale IP-nya di table outlets
Saat monitoring, pusat connect on-demand ke outlet yang dipilih
Gunakan connection pool + timeout
Error handling kuat (jika Tailscale down → tampilkan status "Outlet Offline")

Semua table utama pakai id UUID 16 binary PRIMARY KEY
Buat function helper generate_uuid() dan generate_kode()
Buat DB Manager class yang clean untuk SQLite & MariaDB
Query akan jauh lebih mirip antar database

Pakai BINARY(16) bukan CHAR(36) untuk UUID (hemat space & lebih cepat).
Buat index yang tepat di kolom yang sering dicari (transaction_id, outlet_id, tanggal, product_id).
Tabel transaction_details akan jadi tabel terbesar → ini yang paling penting dioptimasi.
